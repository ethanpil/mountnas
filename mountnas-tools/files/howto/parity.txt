# SnapRAID parity protection
1) Mount the parity disk like any data disk (see: nas howto disks), e.g.:
     UUID=<uuid>  /mnt/parity1  xfs  rw,noatime,nofail  0 2
   The parity disk must be AT LEAST as large as your largest data disk
   (nas status checks this once configured).
2) Edit /etc/snapraid.conf:
     parity  /mnt/parity1/snapraid.parity
     content /mnt/nasdata/snapraid.content
     content /mnt/disk1/snapraid.content
     data d1 /mnt/disk1
     data d2 /mnt/disk2
     exclude *.unrecoverable
     exclude /tmp/
   Keep /mnt/nasdata OUT of the array (it is the system disk).
3) First sync:   snapraid sync
4) Schedule sync + scrub in cron (crontab -e), mailing the report
   (configure mail first: nas howto mail):
     0 3 * * *  snapraid sync 2>&1 | mail -s "snapraid sync $(hostname)" you@example.com
     0 5 * * 0  snapraid scrub -p 8 2>&1 | mail -s "snapraid scrub $(hostname)" you@example.com
5) nas commit
